# CET-6 Input Pool

这些文件是钉钉提醒和后续输入型学习推送的数据源。

## 文件说明

- `words.tsv`：单词 + 中文释义
- `sentences.tsv`：英文句子 + 中文释义
- `expressions.tsv`：常用表达 + 中文释义
- `writing-lines.tsv`：写作可复用句 + 中文
- `translation-lines.tsv`：翻译训练句对照
- `listening-paraphrases.tsv`：听力常见同义替换输入

## 格式

每行一条，使用制表符（TAB）分隔：

`英文<TAB>中文`

## 目标

- 避免提醒内容重复
- 让提醒从固定脚本内容升级为可持续扩充的数据源
- 后续可以继续补充写作句、翻译句、听力高频表达
- 让输入池服务于 CET-6 主线弱项（听力、写作、翻译），而不是无边界堆料

## 维护原则

### 1. 优先级

新增素材时，优先顺序固定为：

1. 能直接服务 30 分钟严格监督型训练的句子 / 表达
2. 能反复出现在写作、翻译、听力复盘中的高频结构
3. 能回收自 `starter-practice-packs.md`、真实打卡复盘、弱项追踪的稳定表达
4. 纯装饰性、低复用、偏阅读炫技的句子放后

### 2. 去重与收口规则

- 发现中英文几乎同义、只是轻微改写的条目时，不重复保留两份
- 同一语义若已有更自然、更通用版本，优先保留更适合 CET-6 输出训练的版本
- 单次 cron 内若只是补了近义改写，不应继续机械扩容；应优先转向筛选、合并、标记高价值条目
- 若新增内容仍主要来自同一批微训练题包，连续扩充 2-3 轮后应优先做一次整理，而不是无限追加

### 3. 来源优先级

- 第一来源：真实用户打卡、错题复盘、评分反馈中反复出现的表达缺口
- 第二来源：`index/starter-practice-packs.md` 中已稳定可复用的句型
- 第三来源：为 Discord 主学习通道补位所需的最小兜底素材
- 未经核验的外部资料，不直接写入输入池核心文件

### 4. 何时新增，何时整理

优先新增的情况：
- 某个弱项模块明显素材不足，已经影响开练
- 新句子能直接填补当前题包或评分复盘中的固定缺口
- 来自真实打卡数据，且重复出现 2 次以上

优先整理的情况：
- 同类句型已经连续补了多轮
- 文件开始出现语义重复或只是措辞轻微变化
- 训练通道已经能正常开练，但输入池还在无差别变厚

### 5. 与主线的关系

- `writing-lines.tsv`、`translation-lines.tsv`、`expressions.tsv` 的扩充，目标是支撑弱项训练和复盘，不是制造“资料很多”的假繁荣
- 真正的资料主线仍是：真实听力样本核验、真实打卡落库、基于真实数据收紧训练口径
- 当输入池可支撑日常开练后，后续应更多依赖真实打卡数据做筛选与淘汰，而不是只做加法
