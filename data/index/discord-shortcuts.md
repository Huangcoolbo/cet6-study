# Discord CET-6 快捷入口返回模板（第一版）

## 目的

把 Discord 主学习通道里的常用快捷入口先固定成可复用返回模板，避免每次都临时组织口径。

本文件当前只覆盖三个高频入口：
1. `查看计划`
2. `查看弱项`
3. `今日复盘`

---

## 适用前提

这些快捷入口默认服务于：
- 30 分钟日常训练
- 严格监督型互动
- 弱项优先（听力 / 写作 / 翻译）
- 不伪造训练记录；若当天没有真实数据，就明确说没有

---

## 一、快捷入口：查看计划

### 触发词
- `查看计划`
- `今日计划`
- `今天练什么`

### 返回目标
让用户一眼看到：
- 今天主攻什么
- 为什么这样排
- 具体做哪三步
- 做完后怎么交

### 标准返回模板

```text
【今日六级计划】
今日主攻：听力 / 写作 / 翻译
原因：根据当前弱项与最近表现，今天先抓 ______。

1. 主任务（10-15 分钟）
- 

2. 补强任务（8-10 分钟）
- 

3. 输出回收（5 分钟）
- 

【提交要求】
请按以下格式提交，不接受“做完了”式回复：
- 完成状态
- 任务1结果
- 任务2结果
- 输出内容
- 今日最卡点
- 自己打分（__/20）
- 明天想继续的点
```

### 固定生成规则

1. 优先从 `baseline.md` 与 `weak-point-tracker.md` 判断主攻模块
2. 若当天已有训练中断记录，优先补断点，不换新主线
3. 阅读只作为保温，不单独占满当天主任务
4. 若无真实新数据，默认按“听力优先，写作/翻译轮换补强”下发

### 当前默认计划骨架（无新数据时）

```text
【今日六级计划】
今日主攻：听力
原因：听力仍是主弱项，且最适合用短时高反馈方式持续拉分。

1. 主任务（12 分钟）
- 做 1 轮听力关键词预判 + paraphrase 识别

2. 补强任务（8 分钟）
- 做 1 轮中译英拆句，重点检查语序和主谓结构

3. 输出回收（5 分钟）
- 用 2-3 句英文总结今天最容易听漏或翻错的点

【提交要求】
按固定格式回传，不接受只说“做完了”。
```

## 二、快捷入口：查看弱项

### 触发词
- `查看弱项`
- `我的弱项`
- `现在最该补什么`

### 返回目标
让用户快速看到：
- 当前主弱项排序
- 最近是否重复出错
- 下一次复测该盯什么

### 标准返回模板

```text
【当前弱项】
第一弱项：
- 模块：
- 问题：
- 错误类型：
- 严重度：
- 是否重复：
- 下次复测：

第二弱项：
- 模块：
- 问题：
- 错误类型：
- 严重度：
- 是否重复：
- 下次复测：

【当前判断】
今天训练优先先补 ______，暂不把阅读当主战场。
```

### 读取顺序

1. 先读 `index/weak-point-tracker.md`
2. 若记录不足，再回看 `daily-logs/2026-03.md` 的“今日主要错误”
3. 若仍无真实数据，则回退到 `baseline.md` 的基础画像

### 无足够真实数据时的默认返回

```text
【当前弱项】
第一弱项：
- 模块：听力
- 问题：真实错因数据仍不足，先按已知主弱项处理
- 错误类型：待补真实训练记录
- 严重度：3
- 是否重复：待判断
- 下次复测：下一次听力训练

第二弱项：
- 模块：写作 / 翻译
- 问题：当前缺首轮真实提交数据，先按既有画像维持第二梯队
- 错误类型：待补真实训练记录
- 严重度：2
- 是否重复：待判断
- 下次复测：听力首轮记录落地后再细分

【当前判断】
现阶段先不要纠结展示花样，先用真实训练把弱项证据补出来。
```

## 三、快捷入口：今日复盘

### 触发词
- `今日复盘`
- `今天复盘`
- `看下今天情况`

### 返回目标
让用户快速看到：
- 今天到底算完成了没有
- 分数是多少
- 最大问题是什么
- 明天必须接什么动作

### 标准返回模板

```text
【今日六级复盘】
日期：YYYY-MM-DD
完成判定：已完成 / 部分完成 / 未完成
总分：__/20

【今天做了什么】
- 

【最大问题】
1. 
2. 
3. 

【一句话复盘】

【明日最小动作】

【是否回写弱项】
需要 / 暂不需要
```

### 读取顺序

1. 优先读取 `daily-logs/2026-03.md`
2. 若已存在评分，再对照 `discord-scoring-review-format.md`
3. 若当天还没有真实训练记录，就明确返回“今日暂无可复盘数据”

### 当天无真实记录时的返回模板

```text
【今日六级复盘】
日期：YYYY-MM-DD
完成判定：暂无记录
总分：暂无

【今天做了什么】
- 还没有可用的真实训练提交

【最大问题】
1. 今天没有留下训练结果，无法复盘
2. 没有输出内容，就无法定位弱项
3. 没有分数，就无法判断训练质量

【一句话复盘】
今天最大的问题不是做错，而是还没有形成可复盘数据。

【明日最小动作】
直接发送“开始今日六级”，至少完成一轮 30 分钟训练并按固定格式提交。

【是否回写弱项】
暂不需要
```

## 四、统一口径

### 1. 不伪造数据
- 没有真实训练记录时，不假装有分数、有错误、有进步
- 允许返回“暂无记录”，不允许编造复盘内容

### 2. 先结论，后解释
- 快捷入口不是长篇教学，优先给结果
- 结果必须能直接引导下一步行动

### 3. 严格监督，不做空泛安慰
- `查看计划` 要具体到动作
- `查看弱项` 要具体到模块和问题
- `今日复盘` 要具体到完成判定和明日最小动作

### 4. 与主入口协同
- 若用户看完 `查看计划` 后准备开练，直接衔接 `开始今日六级`
- 若用户看完 `查看弱项` 后想专项训练，优先导向 `今日听力` / `今日写作` / `今日翻译`
- 若用户看完 `今日复盘` 后需要继续补做，直接给出下一轮最小动作

Discord 主学习通道现在已具备：
- 主入口：`开始今日六级`

3. 再决定是否补“查看本周进度”一类更上层入口
