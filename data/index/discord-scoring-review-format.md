# Discord CET-6 每日评分与复盘格式（第一版）\n\n## 目的\n\n把 Discord 主学习通道里的“用户提交后怎么判定、怎么评分、怎么复盘、怎么给次日动作”固定下来，避免每天重新组织口径。\n\n本文件只解决**每日闭环**，不替代周复盘。\n\n---\n\n## 一、适用场景\n\n- 用户在 Discord 发送 `开始今日六级` 后完成当日任务\n- 用户按 `discord-study-flow.md` 的固定提交格式回传结果\n- 系统需要给出严格监督型评分、指出关键问题、安排次日最小动作\n\n---\n\n## 二、固定反馈顺序\n\n每日反馈必须按以下顺序输出，不跳步、不合并：\n\n1. **完成判定**\n2. **20 分制评分**\n3. **关键问题 1-3 条**\n4. **今日复盘一句话**\n5. **明日最小动作**\n6. **是否需要写入弱项追踪**\n\n> 原则：先判定，再打分，再指出问题，最后给下一步。不能只给鼓励，不给结论。\n\n---\n\n## 三、完成判定口径\n\n### 1. 已完成\n满足以下条件中的大部分：\n- 主任务和补强任务都实际作答\n- 有真实输出内容\n- 有最卡点说明\n- 能看出不是敷衍提交\n\n### 2. 部分完成\n常见情形：\n- 只完成主任务，未完成补强任务\n- 有输出，但明显不完整\n- 提交格式不完整，但仍能看出做过\n\n### 3. 未完成\n常见情形：\n- 只有“做了/没做完”之类空话\n- 没有真实输出\n- 几乎无法判断用户做了什么\n- 明确中断且未补交\n\n---\n\n## 四、20 分制评分细则\n\n沿用 `daily-check-template.md` 的四维结构，每项 5 分，总分 20。\n\n### A. 认真程度（0-5）\n看投入是否真实。\n\n- 5：完整、具体、能看出认真做\n- 4：总体认真，但细节略粗\n- 3：基本完成，但明显应付\n- 2：敷衍痕迹重\n- 1：几乎没投入\n- 0：无有效内容\n\n### B. 完成度（0-5）\n看任务是否做全。\n\n- 5：主任务、补强任务、输出都完成\n- 4：主体完成，个别缺口\n- 3：只完成一半左右\n- 2：只碰了一部分\n- 1：基本没完成\n- 0：未完成\n\n### C. 准确率 / 质量（0-5）\n看答案质量，而不是看是否辛苦。\n\n- 5：错误少，质量稳定\n- 4：有问题，但不影响主体表达/理解\n- 3：存在明显错误，需要纠正\n- 2：问题较多，影响训练价值\n- 1：质量很差\n- 0：无法评估\n\n### D. 改正质量（0-5）\n看是否能识别并修正问题。\n\n- 5：能主动发现关键问题并纠正\n- 4：在提示下能较好修正\n- 3：能改一点，但不彻底\n- 2：知道有问题，但不会改\n- 1：拒绝或回避修正\n- 0：没有改正动作\n\n---\n\n## 五、标准反馈模板\n\n```text\n【今日六级评分】\n完成判定：已完成 / 部分完成 / 未完成\n总分：__/20\n- 认真程度：__/5\n- 完成度：__/5\n- 准确率/质量：__/5\n- 改正质量：__/5\n\n【关键问题】\n1. \n2. \n3. \n\n【一句话复盘】\n\n【明日最小动作】\n\n【弱项追踪】\n需要 / 暂不需要\n若需要：模块 + 问题 + 错误类型 + 严重度 + 是否重复\n```\n\n---\n\n## 六、不同完成状态下的反馈要求\n\n### 若“已完成”\n- 允许肯定，但不能只夸\n- 必须指出仍然最影响提分的 1-2 个问题\n- 明日动作要更聚焦，不要泛泛而谈\n\n### 若“部分完成”\n- 先明确缺口在哪\n- 给分要体现惩罚，不做“完成一半也差不多”的模糊处理\n- 明日动作优先补齐断点，而不是换新题\n\n### 若“未完成”\n- 直接判定，不兜圈子\n- 必须指出未完成的具体表现\n- 明日动作要缩小到可执行最小单元，但不能取消训练要求\n\n---\n\n## 七、弱项追踪触发规则\n\n出现以下情况之一，就应该回写 `weak-point-tracker.md`：\n\n1. 同类错误连续出现 2 次及以上\n2. 错误直接命中听力 / 写作 / 翻译主弱项\n3. 错误严重到明显影响得分\n4. 用户自己提到“今天最卡点”且该问题具备复发风险\n\n### 推荐回写字段\n- 日期\n- 模块\n- 问题\n- 错误类型\n- 严重度（1-3）\n- 是否重复\n- 下次复测\n\n---\n\n## 八、与现有文件的对接\n\n### 每日评分\n优先对照：\n- `index/daily-check-template.md`\n- `daily-logs/2026-03.md`\n\n### 弱项回写\n优先对照：\n- `index/weak-point-tracker.md`\n\n### 周复盘聚合\n后续汇总到：\n- `reviews/2026-W11-weekly-review.md`\n\n---\n\n## 九、当前最小落地范围\n\n本文件当前已固定：\n- 每日评分顺序\n- 20 分制打分口径\n- 标准反馈模板\n- 弱项追踪触发规则\n\n本文件暂未覆盖：\n- 听力 / 写作 / 翻译三类专项的细分评分差异\n- 自动生成周摘要的格式\n- 低分连续多日时的升级干预策略\n\n---\n\n## 十、当前结论\n\nDiscord 主学习通道现在已经不只具备“开练入口 + 提交格式”，还具备了“提交后如何严格评分与复盘”的固定口径。\n\n下一步更适合补的是：\n1. Discord 听力专项互动流程\n2. Discord 翻译专项互动流程\n3. Discord 写作专项互动流程\n\n优先顺序建议先听力，再翻译，再写作。\n
