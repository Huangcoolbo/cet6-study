# CET-6 Data Library

这个目录用于沉淀 CET-6 学习资料，目标是把真题、答案、听力原文、写作与翻译训练材料整理成可长期复用的数据源。

## 当前目录结构（已存在）

- `index/` 总索引、任务板、跟踪器、部分项目运行状态
- `input/` 结构化训练输入池，供提醒链路和后续训练流程消费
- `sources/` 原始来源清单与可信度评估

## 规划中的资料目录

下面这些目录属于仓库的目标结构；随着资料逐步入库会继续补齐：

- `papers/` 真题正文或试卷文件
- `answers/` 参考答案与解析
- `transcripts/` 听力原文、听力材料说明
- `writing/` 写作题目、范文、句型模板、批改记录
- `translation/` 翻译题、参考译文、常见问题
- `reviews/` 周复盘、阶段复盘、资料质量复盘

## 重点说明

### `index/`
这里不只是“索引目录”，还会保存：
- 资料编目
- 学习任务板
- 薄弱点跟踪
- 阶段评估规则
- 提醒链路共享状态（如 `dingtalk-state.json`）

### `input/`
这里保存可以被自动化流程直接消费的结构化输入，例如：
- 单词
- 句子
- 常用表达
- 写作句料
- 翻译句料
- 听力同义改写

这些文件既服务于提醒推送，也为后续训练、抽查和复习流程提供输入。

## 资料分级

- `A`：高可信，来源稳定，可直接用于训练
- `B`：可用，但需人工交叉验证
- `C`：仅作线索，不直接作为核心训练材料

## 命名规则

推荐文件名：

`<year>-<month>-set<1|2|3>-<type>.md`

示例：
- `2023-06-set1-paper.md`
- `2023-06-set1-answers.md`
- `2023-06-set1-transcript.md`

## 当前策略

1. 先建立目录和索引
2. 先沉淀来源，再逐步下载或整理正文
3. 优先补听力原文、答案、解析
4. 结构化输入先服务提醒与轻量训练，再逐步扩展为更完整训练数据
5. 所有资料尽量先入库，再进入后续训练流程
