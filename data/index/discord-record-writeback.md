# Discord 真实打卡回写规则（第一版）

## 目的

把 Discord 主学习通道里**一次真实训练完成后**，应该如何回写到 `daily-logs/`、`weak-point-tracker.md`、周复盘文件的顺序固定下来。

这个文件解决的不是“怎么开练”，而是：
1. 用户一旦真的交了训练结果，哪些文件要更新
2. 先写什么，后写什么
3. 什么情况下只写日记录，什么情况下还要写弱项/周复盘

避免后续真实数据出现后，记录散落在不同文件里、口径不一致。

---

## 适用场景

当 Discord 中出现以下任一情况时，读取本文件：
- 用户按 `【今日六级提交】` 格式提交了真实训练结果
- 用户虽然没完全按模板交，但能确认当天确实完成了部分训练
- 系统已经给出当日完成判定、评分、关键问题与明日最小动作

若只是下发任务、尚无真实提交，不使用本文件。

---

## 回写总原则

1. **先判定，再回写**：没有完成判定与基本评分，不急着写文件
2. **先日记录，再弱项，再周汇总**：避免直接跳到周级、却没留下日级证据
3. **只写真实发生的内容**：没有做过的任务、不确定的错误，不补写
4. **最小够用**：先保证闭环，再逐步丰富细节，不追求一次写满所有文件
5. **同一事实只记一遍主落点**：日级细节主要落 `daily-logs/`，周级只做聚合，不重复抄全文

---

## 固定回写顺序

### 第一步：回写当日日志（必做）
优先更新：
- 当前月份对应的 `D:/Bo/cet6-data/daily-logs/YYYY-MM.md`（例如 `D:/Bo/cet6-data/daily-logs/2026-03.md`）

至少写入：
- 日期
- 状态（已完成 / 部分完成 / 未完成 / 中断）
- 今日任务
- 用户回答记录（保留关键产出，不必全文照抄）
- 今日评分（20 分制）
- 今日主要错误
- 明日重点

### 第二步：判断是否回写弱项追踪（条件触发）
符合以下任一情况，再更新：
- `D:/Bo/cet6-data/index/weak-point-tracker.md`

触发条件：
1. 同类错误连续出现 2 次及以上
2. 问题命中听力 / 写作 / 翻译主弱项
3. 该错误足以明显影响分数
4. 用户自述的“今日最卡点”具有复发风险

回写字段：
- 日期
- 模块
- 问题
- 错误类型
- 严重度（1-3）
- 是否重复
- 下次复测

### 第三步：等待周复盘聚合（不要求每日都写）
周级文件：
- 当前周对应的 `D:/Bo/cet6-data/reviews/YYYY-Www-weekly-review.md`（例如 `D:/Bo/cet6-data/reviews/2026-W11-weekly-review.md`）

规则：
- 日常训练完成后，不要求立刻逐条回写周复盘
- 周复盘在形成一周真实记录后统一聚合
- 若本周仍无足够真实数据，则保留“待首轮真实打卡数据”状态，不伪造周统计

---

## 日志回写的最小模板

当日完成后，优先按下列结构补到月日志：

```markdown
## YYYY-MM-DD
- 状态：已完成 / 部分完成 / 未完成 / 中断
- 今日任务：
- 用户回答记录：
- 今日评分：__/20
- 今日主要错误：
- 明日重点：
```

### 压缩原则

- `今日任务`：保留任务目标与模块，不必重复整段指令
- `用户回答记录`：只保留可证明训练真实发生的关键产出
- `今日主要错误`：优先写最影响提分的 1-3 个，不堆细枝末节
- `明日重点`：写成可直接续练的最小动作，而不是大而空的计划

## 与 Discord 评分层的衔接

回写前，应先完成：
- `完成判定`
- `20 分制评分`
- `关键问题 1-3 条`
- `一句话复盘`
- `明日最小动作`

对应读取：
- `D:/Bo/cet6-data/index/discord-scoring-review-format.md`
- `D:/Bo/skills/cet6-coach/references/discord-scoring-review.md`

本文件只处理**评分之后如何落库**。

## 专项训练的附加说明

### 听力
若当日是听力训练：
- 先照常写 `daily-logs/`
- 若暴露的是“听不出同义改写 / 抓不到信号词 / 跟不上转折”这类可复发问题，再写 `weak-point-tracker.md`
- 不因为做了听力训练，就自动把资料样本状态写成 `trained`

### 写作
若当日是写作训练：
- 日志里保留最关键的产出句或段落问题
- 若错误集中在结构、时态、主谓一致、句式单一，可回写弱项

### 翻译
若当日是翻译训练：
- 日志里保留最典型的中式直译或拆句问题
- 若同类语序/搭配错误反复出现，再回写弱项

---

## 不该做的事

- 没有真实提交时，提前补写 `daily-logs/`
- 仅因为“今天本该学”就把状态写成已完成
- 把用户一句含糊反馈包装成完整训练记录
- 当天刚写过日记录，又把同样内容整段复制到周复盘
- 因为做了练习，就顺手上调资料样本状态

---

## 当前结论

Discord 主学习通道当前缺的已不是“怎么开练”，而是**真实打卡后的统一落库顺序**。

本文件已把第一版顺序固定为：
1. 先评分与判定
2. 必写 `daily-logs/`
3. 条件触发时写 `weak-point-tracker.md`
4. 周复盘等待真实周数据统一聚合

后续等首轮真实打卡数据出现后，再继续收紧字段、去掉当前这版里的保守兜底口径。
